# Security Policy

## Supported Versions

This project is in an early open-source foundation phase. Security fixes are only supported on the latest state of `main`.

## Reporting a Vulnerability

Please do not open public GitHub issues for suspected vulnerabilities.

Report security concerns privately by:

1. Using GitHub's private vulnerability reporting flow for this repository if it is enabled.
2. If private reporting is not available, contacting `@Mohit4022-cloud` on GitHub and requesting a private disclosure channel.

When reporting an issue, include:

- affected route, API, or file path
- reproduction steps
- impact assessment
- any suggested mitigation or fix

You can expect an initial acknowledgement as soon as the maintainer is able to review the report. Please keep vulnerability details private until a fix or mitigation is available.
